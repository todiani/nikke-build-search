# 사용자 개발 지침 및 선호도 (USER_GUIDELINES.md)

이 파일은 사용자의 요청 사항, 선호하는 UI/UX 패턴, 금지 사항 등을 기록하여 개발 시 일관성을 유지하기 위한 문서입니다. 사용자의 명시적인 변경 요청이 있기 전까지 이 지침을 최우선으로 준수합니다.

## 1. UI/UX 지침
- **추천 조합(Meta) 관리**: 조합 추가 및 수정 시, 텍스트 입력 대신 **인라인 레이어(모달)**를 사용하며, 니케 선택 시 시스템의 **니케 선택창(Selector)**을 호출하여 직관적으로 선택할 수 있어야 함.
- **폰트 크기**: 추천 조합 및 메타 데이터 표시 시 가독성을 위해 큰 폰트 크기를 유지함.
- **카테고리 명칭**:
    - Anomaly -> **이상 개체 요격전 (특특요)**
    - Special Interception -> **이상 개체 요격전**
- **메뉴 간소화**: 🤖 자동 매칭, 보유 니케 조합만 보기 등 불필요한 필터 메뉴는 제거하고 CRUD(추가/수정/삭제) 기능에 집중함.

## 2. 데이터 관리 지침
- **데이터 무결성**: 모든 니케 데이터는 `nikke_db.json`의 ID와 이름을 기준으로 관리하며, `future_meta.ts`와 같은 별도 파일 기반의 관리는 금지함.
- **영속성**: 모든 변경 사항(태그 추가, 메타 팀 추가 등)은 즉시 서버의 `nikke_db.json`에 저장되어 새로고침 후에도 유지되어야 함.
- **인덱싱**: 데이터 규모 증가에 대비하여 태그 검색 등은 메모리 내 인덱싱 방식을 사용하여 성능을 최적화함.

## 3. 금지 사항
- 데이터 무결성 감사(Audit) 탭 생성 금지.
- 게스트 니케(GUEST_NIKKES)와 같은 하드코딩된 외부 데이터 사용 지향 (DB 데이터 우선).
