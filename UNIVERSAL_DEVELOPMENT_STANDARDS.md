# 🌐 Universal Development Standards (범용 프로젝트 개발 표준 가이드)

이 문서는 고품질 소프트웨어 개발과 AI 에이전트와의 효율적인 협업을 위해 설계된 범용 개발 표준 가이드입니다. 어떠한 기술 스택(React, Python, Node.js 등)에서도 적용 가능한 핵심 원칙을 담고 있습니다.

---

## 1. 🏗️ 아키텍처 및 데이터 무결성 (Data Integrity)

### 1.1. Single Source of Truth (SSOT)
- 모든 애플리케이션 상태는 하나의 신뢰할 수 있는 원본(DB 또는 전역 상태)을 가져야 합니다.
- **원칙**: 데이터의 중복을 피하고, 변경 발생 시 원본을 수정하면 모든 연결된 UI/로직이 즉시 갱신되어야 합니다.

### 1.2. 위치 보존의 원칙 (Position Integrity)
- 배열이나 객체 내의 데이터 순서 및 인덱스는 비즈니스 로직상 의미가 있을 경우 절대 임의로 변경하지 않습니다.
- **예시**: 오버로드 장비의 1, 2, 3번 슬롯 데이터 업데이트 시, 유효한 값만 앞으로 당겨서 저장하는 행위를 금지합니다. 빈 슬롯은 명시적으로 `null` 또는 기본값으로 유지합니다.

### 1.3. 데이터 정규화 (Normalization)
- 외부 데이터(스크래핑, API 등)를 내부 시스템으로 유입할 때 반드시 정규화 과정을 거칩니다.
- **표준화**: 명칭, 단위, 날짜 형식 등을 프로젝트 공통 상수를 활용하여 변환합니다.

---

## 2. 🤖 AI 에이전트 협업 지침 (AI Collaboration)

### 2.1. 컨텍스트 파일 활용 (`AGENTS.md`)
- 프로젝트 루트에 AI 에이전트 전용 가이드 파일(`AGENTS.md`)을 유지합니다.
- 이 파일에는 기술 스택, 디렉터리 구조, 보안 제한 사항, 그리고 **절대 어겨서는 안 될 금기 사항**을 명시합니다.

### 2.2. 실시간 동기화 (Real-time Sync)
- 모든 데이터 작업(CRUD)은 비동기 처리가 완료된 후 즉시 UI에 반영되어야 합니다.
- **패턴**: `async/await`를 사용하여 데이터 저장이 완료된 것을 보장한 후 상태를 갱신하거나 새로고침 이벤트를 발생시킵니다.

---

## 3. 🛠️ 품질 관리 및 안정성 (Quality & Reliability)

### 3.1. 자동 백업 및 복구 (Backup & Recovery)
- 중요한 데이터 파일(JSON, SQLite 등)을 직접 수정하는 작업 전후에는 반드시 자동 백업을 생성합니다.
- **파일명**: `[filename]_[ISO_TIMESTAMP].json` 형식을 권장합니다.

### 3.2. 비동기 처리의 대기 시간 (Race Condition 방지)
- 외부 시스템이나 브라우저 자동화(Puppeteer, Playwright 등) 시 UI가 완전히 로드될 때까지 충분한 대기 시간을 가집니다.
- **기준**: 단순 고정 대기(`setTimeout`)보다는 특정 요소의 출현을 기다리는 `wait_for` 패턴을 우선 사용합니다.

---

## 4. 🎨 UI/UX 일관성 (UI Consistency)

### 4.1. 컴포넌트 공유 (Component Reuse)
- 동일한 목적을 가진 기능(예: 데이터 편집, 상세 조회)은 반드시 동일한 컴포넌트나 로직을 공유하여 코드 중복을 최소화합니다.

### 4.2. 가독성 중심의 디자인
- 텍스트 위주의 정보 나열보다는 이모지, 색상 코드, 시각적 그래프를 적극 활용하여 사용자가 정보를 즉각 파악할 수 있도록 합니다.

---

## 5. 📝 문서화 규칙 (Documentation)

- **코드 주석**: 복잡한 로직은 '왜(Why)' 이 코드가 필요한지에 대한 설명을 주석으로 남깁니다.
- **커밋 메시지**: 작업 단위별로 명확한 접두사(`feat:`, `fix:`, `refactor:`)를 사용합니다.
- **언어**: 프로젝트의 주 사용자의 언어(예: 한국어)로 일관되게 작성합니다.
